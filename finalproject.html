『 ☆Background☆ 』

This idea was sparked when I saw my mum gardening but said that it has been hard to make them recently as her fingers are too big and not precise enough, and it was really dirty.

Hence, I realized that when it came to detailed tasks, a supplementary helper specialized in touching the details would be helpful when our own hands often lack the precision and detail to make smaller scale projects.

After brainstorming, I decided that a robotic arm could act as a small helper around the house for small tasks and to aid my mum's projects.
『 ☆Ideation☆ 』

Name: M.A.R.C.H (The “Multipurpose Artificial Reaching Clasping Hand”)

Object: A robotic arm with embedded programming using Arduino

Function: To aid in picking up and organising items for small scale daily tasks

Purpose: To help users improve their quality of life and work, especially for small details hard to reach on our own.

Target Audience: Households and hobbyists.

The name of the project is derived from the purpose of the project, to create a mini helping tool around the house, almost as if the user can equip an extra set of hands to aid in work. The arm can also engage in messier tasks that we do not feel keen on doing, incentivising users to embark on more projects. In addition, the novelty of the item also adds an element of play to the task, making daily activities more fulfilling.

On the right shows a sketch of my project and how it works, done using Gimp and Procreate.

『 ☆Step 1: Computer Aided Design☆ 』

Engraving Design

I wanted to use a nature-themed design, as it would pay homage to the inspiration for this design, which was to aid in gardening.

I imported pictures of plants and flowers into Inkscape's bitmap tool and used it to create vector drawings of flowers that I could sleek designs of them into the arm.

Some of the designs (the thicker ones) were also done in raster for engraving.

  1. Imported nature-themed images into Inkscape's bitmap tool
  2. Created vector drawings of flowers for arm designs
  3. Used raster for engraving thicker designs

Designing the Arm

I first started to draft my design out, with the specifications of each component.

I also took note of the dimensions and sketched out each element I needed to tie together and started modeling the components into an arm.

I took some inspiration from Pinterest and using the joint control feature on Fusion 360, I was able to ensure the arm was movable and could be moved by the rotating of individual servos.

  1. Drafted design with component specifications
  2. Noted dimensions and started modeling components
  3. Used Fusion 360's joint control for arm movability

    4. >

Designing the Base

I used techniques from designing a finger joint box here and made the base large enough to both host and place misc parts of the arm in.

  1. Applied finger joint box techniques for base

    2. Some of the designs (the thicker ones) were also done in raster for engraving.

  2. Integrated Arduino board into the top of the base
  3. Incorporated floral design elements on the base
  4. Exported arm and base files as DXF for laser cutting

Designing the Controller

I then designed the controller box to host the joysticks and the breadboard, but this was to be 3D printed so we did not have to export it as a DXF.Instead, we export it to cura for slicing (done in the step 3 for 3d printing).

  1. Used paramteters to create the dimensions for the controller
  2. Created controller box design for joysticks and breadboard
  3. Implemented snapping mechanism for box lid
  4. Added two 25mm holes for joystick placement
『 ☆Step 2: Laser Cut☆ 』

DXF for Laser Cutting

Fusion 360 is used for designing and creating the design. The design is then being laser cut out of 4mm plywood.

  1. Create new sketches to save each piece as a DXF file since they weren't drawn in one sketch initially.
  2. Use a DXF conversion tool and start converting each face to DXF.
  3. Repeat the steps for all unique pieces.
  4. Right-click on the sketch and select ‘Save as DXF’.
  5. Prepare the DXF files for vector cutting.

Cutting

For laser cutting, I utilized CorelDraw to process the DXFs.

  1. Ensure the line weight is set to Hairline and the color is RGB (255, 0, 0).
  2. Go into preferences and adjust settings as needed.
  3. Note: Despite the higher power and slower speed, the cut didn't penetrate the first time, potentially due to a dirty lens.

>

Engraving

For the parts not set to Hairline width, I aimed for an engraved design instead of a full cut.

  1. Set the color for these parts to RGB (0,255,0).
  2. Ensure the color designation results in raster engraved designs rather than cuts.

>
『 ☆Step 3: 3D Print☆ 』

Cura Slicing

This step aimed to optimize the STL file for printing on a specific printer.

I used Prusa printers from the library and started with a print duration of up to 7 hours with a 20% infill.

After optimizing by adjusting piece positions and changing infill to 10%, the print time decreased to 4 hours.

  1. Ensure the STL file is optimized for the chosen printer.
  2. Use Prusa printers from the library as the chosen printer.
  3. Original print took up to 7 hours with 20% infill.
  4. Optimize by adjusting piece positions to be flat on base.
  5. Change infill to 10% to reduce print time.

    6. >

Printing

Once the settings were finalised, I initiated the print process.

>

After 4 hours, I returned and retrieved the completed print.
『 ☆Step 4: Embedded Programming☆ 』

Working with Robotic Arm Code

This section discusses the challenges and solutions related to the code for the robotic arm.

Originally, the code wasn't functioning as expected on the prototype. However, the provided code snippets below are the final and functional versions. Additional details about the trial and error process are also discussed in the prototyping section.

Circuit Testing

Unfortunately, TinkerCAD does not have joysticks, hence my code had to be tested out physically (next section). Howver, to prep myself, I got some circuit diagrams ready and component functions:

  • Joystick 1: Arm movement up and down, forward and backward
  • Joystick 2: Base swivel, claw open and close

Finalized Code

Here's the final version of the code:

Servo s1, s2, s3, s4; const int joy1XPin = A1; const int joy1YPin = A0; const int joy2XPin = A3; const int joy2YPin = A2; const int servo1Pin = 6; const int servo2Pin = 9; const int servo3Pin = 10; const int servo4Pin = 11; int joy1XValue, joy1YValue, joy2XValue, joy2YValue; const int servoMinAngle = 0; const int servoMaxAngle = 180; s1.attach(servo1Pin); s2.attach(servo2Pin); s3.attach(servo3Pin); s4.attach(servo4Pin); Serial.begin(9600); } void loop() { joy1XValue = analogRead(joy1XPin); joy1YValue = analogRead(joy1YPin); joy2XValue = analogRead(joy2XPin); joy2YValue = analogRead(joy2YPin); int servo1Angle = map(joy1XValue, 0, 1023, servoMinAngle, servoMaxAngle); int servo2Angle = map(joy1YValue, 0, 1023, servoMinAngle, servoMaxAngle); int servo3Angle = map(joy2XValue, 0, 1023, servoMinAngle, servoMaxAngle); int servo4Angle = map(joy2YValue, 0, 1023, servoMinAngle, servoMaxAngle); s1.write(servo1Angle); s2.write(servo2Angle); s3.write(servo3Angle); s4.write(servo4Angle); Serial.print("Joystick 1 X: "); Serial.print(joy1XValue); Serial.print("\tY: "); Serial.print(joy1YValue); Serial.print("\t\tJoystick 2 X: "); Serial.print(joy2XValue); Serial.print("\tY: "); Serial.println(joy2YValue); delay(20); }

This code successfully controls the robotic arm's actions according to joystick inputs.
『 ☆Step 5: Prototyping☆ 』

This part had the most issues, and the code had to be changed multiple times to accommodate the hardware. Since, joysticks did not exist in TinkerCAD either, the prototype could only be tested by manually uploading the code

Version 1:

This had some very weird responses, and I was using a code for it that I sampled online. The machine would not respond to the joysticks at all, rather it would move in random directions when the joysticks were turned. It almost seemed like the robot had a life of its own!

Troubleshooting:

The issue seemed to be that the code overrode itself in certain areas, and was not compatible with my particular build. For example, my servos were not of the same rotation angle as in the sampled code, and the movement axis in the code was different from what axis I wanted my servos to move. Hence, I decided to start the code from scratch using my own knowledge on Arduino Code.

Troubleshooting:

The issue was that I did not set a delay for the arm, hence its movements were extremely fast causing its over-aggressive swings. Hence, I set a delay of delay(15). Here is the explanation of how the code would work with delays:

                                                        // move pretty slowly, you may want to make this
                                                        // delay longer on a physical Arduino.

Version 3:

After updating the delay in the code, I decided to rewire the robot so it would be less messy. However, this led to a worse issue. The wiring had gone wrong, and the robot was no longer responding to the joysticks at all!

Troubleshooting:

To amend this issue, I borrowed a servo checker from MakerSpace. It turns out that one of the servos was broken, and that the wiring was not very secure around the power source. In addition, the batteries seemed to be quite old.

To fix this, I swapped out the broken servo and the batteries and reconnected the wires that were not connected well.

Here I started relearning arduino while studying case studies and recoding the arm...fighting!!!

Version 4:

After watching a few more videos on coding with Arduino for joystick input and servo output, I was confident that this attempt would work. I tweaked some areas in the code, and alas, the entire arm was working!

Old code with movement issues:


                                                                #include 

                                                                    Servo Servo_X;
                                                                    Servo Servo_Y;
                                                                    Servo Servo_Grip;
                                                                    
                                                                    #define Joy_X A0
                                                                    #define Joy_Y A1
                                                                    #define Joy_SW A2
                                                                    
                                                                    int Joy_Val = 0;
                                                                    boolean state = false;
                                                                    
                                                                    void setup()
                                                                    {
                                                                      Servo_X.attach(3);
                                                                      Servo_Y.attach(5);
                                                                      Servo_Grip.attach(6);
                                                                      pinMode(Joy_SW, INPUT_PULLUP);
                                                                    }
                                                                    
                                                                    void loop()
                                                                    {
                                                                    
                                                                      Joy_Val = analogRead(Joy_X);
                                                                      Joy_Val = map(Joy_Val, 0, 1023, 180, 0);
                                                                      Servo_X.write(Joy_Val);
                                                                    
                                                                      Joy_Val = analogRead(Joy_Y);
                                                                      Joy_Val = map(Joy_Val, 0, 1023, 0, 180);
                                                                      Servo_Y.write(Joy_Val);
                                                                      delay(15);
                                                                    
                                                                      if (digitalRead(Joy_SW) == LOW) {
                                                                    
                                                                        if (state == false) {
                                                                          state = true;
                                                                          Servo_Grip.write(90);
